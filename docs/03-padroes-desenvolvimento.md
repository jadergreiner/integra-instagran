
# Padrões de Desenvolvimento

- TDD e testes unitários com nomes verbosos em português
- Estrutura case-when (dado/quando/então)
- Lint e PEP8 obrigatórios
- Workflow Git: branches feature/*, merge para develop após testes, release protegida, main protegida
- Princípios: YAGNI, KISS, entrega incremental, data-driven
